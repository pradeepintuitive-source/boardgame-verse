# Project Memory: Project Overview (project.md)

## Project Overview
**GameHub** (also referred to in the codebase as **Boardgame-Verse**) is a premium, real-time board gaming platform featuring digital editions of popular classic board and social deduction games. The application currently fully supports two distinct board games:
1. **Mafia**: A social deduction game for 6–12 players where players attempt to find the imposters (Mafia) in their midst.
2. **Monopoly**: A property-trading game for 2–6 players where players roll dice, buy/rent properties, build houses/hotels, and trade to bankrupt their rivals.

## Purpose
GameHub is designed to provide a premium arcade-like experience. It features high-fidelity typography, glowing neon aesthetics, responsive micro-animations, and full support for both online multiplayer play and offline-ready local single-device play with AI.

## Key Features
* **Hybrid Multiplayer Modes**: Support for local single-device pass-and-play, local area network (LAN) mode, and online multiplayer matches.
* **WebSocket Real-Time Broadcasts**: Direct state updates pushed to connected clients using STOMP-over-WebSocket protocols.
* **Autonomous AI Bots**: Configurable AI players built into both the local simulations and the backend match processes to fill empty lobby seats.
* **Interactive In-Game Chat**: Direct messaging, system narration announcements, and private channels (e.g., Mafia role chat).
* **Robust Offline Fallback**: Complete client-side game engines (`mafiaEngine` and `monopolyEngine`) driving local state mutations in Zustand stores if the remote WebSocket server is unavailable.

## Tech Stack

### Frontend & Core Shell
* **Runtime / Compiler**: Vite 7 & Bun (packaged lock file `bun.lock` + fallback node-lock).
* **Framework**: React 19.
* **Routing**: TanStack Router & TanStack Start (routing-tree generated file `routeTree.gen.ts`).
* **State Management**: Zustand 5 & React Query (TanStack Query) v5.
* **Styling**: Tailwind CSS v4 (`@tailwindcss/vite` integrated with Vite config, utilizing the OKLCH color space for vivid neon gradients).
* **Animations**: Framer Motion 12.
* **Component Library**: Radix UI Primitives (Accordion, Dialog, Select, Dropdown Menu, etc.) and Lucide React icons.
* **Network & Sockets**: Axios (REST APIs) + SockJS Client + `@stomp/stompjs` (WebSockets).

### Backend (External)
* **Framework**: Spring Boot (Java).
* **WebSocket API**: Spring WebSocket Messaging (`SimpMessagingTemplate`).
* **Realtime Protocol**: STOMP over SockJS.

## Build and Run Commands

### Development Server
Starts the local development server (typically on port 3000, exposed to local host network):
```bash
npm run dev
```

### Static Production Build
Compiles TypeScript, packages modules, and builds optimized static assets inside `/dist` directory:
```bash
npm run build
```

### Build in Dev Mode
Compiles assets with source-mapping and mode set to development:
```bash
npm run build:dev
```

### Preview Build
Serves the local `/dist` directory contents for review:
```bash
npm run preview
```

### Code Quality Check (Linter)
Analyzes codebase against ESLint configurations:
```bash
npm run lint
```

### Formatting
Applies Prettier styling modifications to all files:
```bash
npm run format
```
